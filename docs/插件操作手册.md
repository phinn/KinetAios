## 一、插件系统总览

KinetAios 插件系统允许通过**零侵入核心架构**扩展 Agent 能力。每个插件可以包含以下组成部分：

| 组件 | 说明 | 是否必需 |
|------|------|----------|
| **工具 (Tools)** | 可被 Agent 主动调用的函数 | 至少一个 |
| **Slash 命令** | 用户输入 `/命令名` 触发的预设工作流 | 可选 |
| **系统提示词** | 注入到 Direct 引擎的领域知识/行为指引 | 可选 |
| **图标** | 插件卡片展示的 SVG 图标 | 可选 |

### 引擎兼容性

| 引擎 | 工具调用 | Slash 命令 | 系统提示词 |
|------|----------|------------|------------|
| **Direct (Kaios)** | ✅ 原生支持 | ✅ | ✅ |
| **Claude Code** | ❌ | ❌ | ✅ (通过 `--append-system-prompt`) |
| **Codex** | ❌ | ❌ | ✅ (通过 prompt 前缀) |

> ⚠️ 插件工具目前仅 Direct 引擎可用。切换到 Claude Code / Codex 后，插件工具不会生效。

### 插件加载位置

| 模式 | 扫描路径 |
|------|----------|
| **开发模式** (`npm start`) | 项目根目录 `plugins/` + `<userData>/plugins/` |
| **生产模式** (打包后) | `<userData>/plugins/` |

## 三、low-altitude（低空经济行业插件）

**定位**: 无人机/eVTOL 运营、低空空域管理、飞行任务规划  
**权限**: `shell` · `web_fetch` · `network`  
**分类**: data

### 3.1 工具详解

#### 🔧 `weather_brief` — 航空气象简报

获取指定坐标的航空气象数据，用于飞行前天气评估。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `lat` | number | ✅ | — | 纬度 (-90 ~ 90) |
| `lon` | number | ✅ | — | 经度 (-180 ~ 180) |
| `hours` | number | ❌ | 12 | 预报时长（小时，最大 48） |

**返回字段**: 风速、风向、能见度、降水概率、云底高、温度

**数据源**: Open-Meteo（免费、无需 API Key），也支持通过环境变量 `LOWALT_WEATHER_API` 指定自定义数据源。

> 💡 **自然语言触发**: "查一下深圳今天的飞行天气"、"经纬度 22.5, 114.0 的气象怎么样"

---

#### 🔧 `airspace_check` — 低空空域/限飞区查询

查询指定坐标和高度是否处于限飞区、管制区或临时禁飞区。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `lat` | number | ✅ | 纬度 (-90 ~ 90) |
| `lon` | number | ✅ | 经度 (-180 ~ 180) |
| `altitude` | number | ✅ | 计划飞行高度（米，真高 AGL） |

**返回字段**: 空域类型（G/W2/C/D/禁飞区）、飞行许可建议（允许/报备/申请/禁止）

> 💡 **自然语言触发**: "这个坐标 120 米高度能飞吗"、"宝安区 300 米是不是管制区"

---

#### 🔧 `flight_plan` — 航线规划

输入起降点和途经点，自动计算总航程、预估飞行时间、电量需求。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `waypoints` | array | ✅ | — | 航点列表，至少 2 个点 |
| ↳ `.lat` | number | ✅ | — | 纬度 |
| ↳ `.lon` | number | ✅ | — | 经度 |
| ↳ `.name` | string | ❌ | — | 航点名称（如"起飞点"） |
| `droneType` | string | ❌ | `generic-multirotor` | 无人机型号 |
| `cruiseSpeed` | number | ❌ | 15 | 巡航速度 (m/s) |
| `batteryCapacity` | number | ❌ | 5000 | 电池容量 (mAh) |

**返回字段**: 总航程 (km)、预估飞行时间 (min)、各航段详情、电量消耗评估、返航余量

> 💡 **自然语言触发**: "帮我规划从深圳到珠海的航线"、"三个航点的巡检路线"

---

#### 🔧 `compliance_check` — 法规合规检查

基于中国民航局 CCAR-92 法规框架，检查飞行场景是否合规。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `scenario` | string | ✅ | 飞行场景（见下表） |
| `droneCategory` | string | ✅ | 无人机类别（见下表） |
| `flightArea` | string | ✅ | 飞行区域类型（见下表） |
| `altitude` | number | ❌ | 飞行高度（米，真高 AGL） |
| `bvlos` | boolean | ❌ | 是否超视距飞行，默认 `false` |

**`scenario` 可选值**:

| 值 | 含义 |
|----|------|
| `inspect` | 巡检（电力/管道/桥梁等） |
| `logistics` | 物流配送 |
| `photography` | 航拍摄影 |
| `agriculture` | 农业植保 |
| `mapping` | 测绘 |
| `emergency` | 应急救援 |

**`droneCategory` 可选值**:

| 值 | 含义 | 重量范围 |
|----|------|----------|
| `micro` | 微型 | < 250g |
| `light` | 轻型 | < 4kg |
| `small` | 小型 | 4 - 15kg |
| `medium` | 中型 | 15 - 116kg |
| `large` | 大型 | > 116kg |
| `evtol` | 载人 | eVTOL |

**`flightArea` 可选值**:

| 值 | 含义 |
|----|------|
| `uncontrolled` | 非管制区（G类） |
| `controlled` | 管制区（C/D类） |
| `airport` | 机场附近 |
| `urban` | 城市上空 |
| `rural` | 郊区/农村 |
| `border` | 边境 |

**返回字段**: 合规检查清单（逐项 ✅/⚠️/🔴）、需要办理的手续列表、总体评估

> 💡 **自然语言触发**: "轻型无人机在城市巡检合规吗"、"植保机在郊区 50 米需要什么手续"

### 3.2 Slash 命令

在对话框输入 `/` 可触发命令补全。三个命令覆盖完整的飞行任务流程：

#### 📋 `/airspace-brief` — 空域态势简报

**用途**: 快速了解某区域的限飞状态、法规要求和天气情况。

**流程**:
```
用户输入坐标/高度/用途
    ↓
并行调用：airspace_check + compliance_check + weather_brief
    ↓
输出值班简报格式（空域状态 + 法规要点 + 气象条件 + 建议）
```

**示例对话**:
```
用户: /airspace-brief
Agent: 请提供以下信息：
  1. 飞行区域坐标（纬度, 经度）
  2. 计划飞行高度（米）
  3. 飞行用途（巡检/航拍/物流...）

用户: 22.5, 114.0, 高度 100 米, 电力巡检
Agent: 📋 空域值班简报
  ✅ 空域类型: G类（非管制空域）
  ✅ 法规: 轻型巡检合规，无需报备
  ⚠️ 气象: 风速 7m/s，接近限值
  → 建议: 可飞行，注意阵风
```

---

#### 📋 `/plan-flight` — 飞行任务一键规划

**用途**: 一站式完成飞行前的全部准备工作。

**流程**:
```
收集参数（起降点 / 高度 / 机型 / 场景）
    ↓
并行查询：weather_brief + airspace_check + compliance_check
    ↓
flight_plan 计算航线
    ↓
综合评估 → 生成《飞行任务书》
```

**输出结构化任务书包含**:
- 气象评估
- 空域状态
- 合规检查结果
- 航线详情（航段、距离、时间）
- 电量评估
- **最终结论**: ✅ 建议飞行 / ⚠️ 有条件飞行 / 🔴 不建议飞行

---

#### 📋 `/inspect-report` — 巡检报告生成

**用途**: 整理巡检数据、识别缺陷、对比历史、输出结构化报告。

**流程**:
```
确认巡检类型 / 区域 / 数据来源
    ↓
读取巡检日志 / 分析照片数据
    ↓
recall_memory 搜索历史记录对比
    ↓
生成结构化巡检报告 → 可选导出为文件
```

**输出报告包含**:
- 巡检概要（时间、区域、设备）
- 发现的问题清单（按严重程度分级）
- 与历史数据对比（趋势分析）
- 处置建议

## 四、echo（示例插件）

**定位**: SDK 验证 / 开发模板  
**用途**: 复制此目录改造即可快速开发新插件

### 工具

#### 🔧 `echo_args` — 参数回显

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `payload` | string | ✅ | 任意字符串，会被原样回显 |

**示例**:
```
用户: 用 echo 工具测试一下 "hello world"
Agent: [调用 echo_args]
  echo: {"payload":"hello world"}
```

### 开发参考

echo 插件的最小结构如下，可作为新插件模板：

```
plugins/
└── echo/
    ├── plugin.json    ← manifest 声明
    └── index.js       ← 工具实现
```

**plugin.json 最小模板**:
```json
{
  "name": "echo",
  "version": "1.0.0",
  "description": "示例插件",
  "tools": "index.js#tools"
}
```

**index.js 最小模板**:
```javascript
module.exports.tools = [
  {
    name: 'echo_args',
    description: '原样返回参数',
    readOnly: true,
    inputSchema: {
      type: 'object',
      properties: {
        payload: { type: 'string', description: '任意字符串' }
      },
      required: ['payload']
    },
    run: async (args) => {
      return 'echo: ' + JSON.stringify(args);
    }
  }
];
```

## 五、插件管理操作

### 5.1 查看插件列表

1. 点击侧边栏 **设置** ⚙️
2. 进入 **插件** Tab
3. 顶部显示统计：总插件数 · 已启用 · 工具总数 · 命令总数
4. 支持按分类筛选、按关键词搜索

### 5.2 查看插件详情

1. 在插件列表中，**点击插件卡片的名称/描述区域**
2. 展开详情面板，显示：
   - 🔧 工具列表（名称 + 描述）
   - Slash 命令列表（名称 + 描述）
   - 权限声明
   - 系统提示词预览（前 500 字）
   - 插件目录路径
   - 错误详情（如有加载失败）
3. 再次点击收起

### 5.3 启用/禁用插件

- 点击插件卡片右侧的 **开关** 切换启用/禁用
- 禁用后插件工具立即从 Agent 可用列表移除
- 加载失败的插件开关不可操作

### 5.4 卸载插件

- 点击插件卡片右侧的 **垃圾桶图标** 🗑
- 确认后从系统中移除
- 仅 `<userData>/plugins/` 下的插件可卸载
- 开发模式下 `plugins/` 目录的插件卸载后重启会重新加载

### 5.5 安装插件

**方式一：拖放安装**
- 将插件文件夹/压缩包拖放到设置页的安装区域
- 自动解压并注册

**方式二：手动安装**
- 将插件文件夹复制到 `<userData>/plugins/`（生产）或项目 `plugins/`（开发）
- 点击 **重载** 按钮刷新插件列表

### 5.6 重载插件

- 点击设置页的 **重载** 按钮
- 热重载所有插件，无需重启应用
- 开发时修改插件代码后点此生效

## 六、开发新插件

### 6.1 目录结构

```
plugins/
└── my-plugin/
    ├── plugin.json          ← 必需，manifest 声明
    ├── index.js             ← 必需，工具实现
    ├── prompt.md            ← 可选，系统提示词
    ├── icon.svg             ← 可选，SVG 图标
    └── commands/            ← 可选，Slash 命令
        ├── cmd1.md
        └── cmd2.md
```

### 6.2 plugin.json 完整字段

```json
{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "插件描述",
  "author": "作者名",
  "category": "office",
  "icon": "icon.svg",
  "engines": ["direct"],
  "permissions": ["shell", "web_fetch"],
  "tools": "index.js#tools",
  "slashCommands": "commands",
  "systemPrompt": "prompt.md"
}
```

**字段说明**:

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | 唯一标识符 |
| `version` | string | ✅ | 语义化版本号 |
| `description` | string | ✅ | 简短描述 |
| `author` | string | ❌ | 作者 |
| `category` | string | ❌ | 分类: `office`/`dev`/`media`/`data`/`system`/`misc` |
| `icon` | string | ❌ | SVG 图标文件路径 |
| `engines` | string[] | ❌ | 支持的引擎，默认 `["direct"]` |
| `permissions` | string[] | ❌ | 权限声明: `shell`/`web_fetch`/`network` |
| `tools` | string | ✅ | 工具入口，格式 `文件.js#导出名` |
| `slashCommands` | string | ❌ | 命令目录路径 |
| `systemPrompt` | string | ❌ | 系统提示词文件路径 |

### 6.3 工具实现规范

```javascript
module.exports.tools = [
  {
    name: 'my_tool',
    description: '工具描述（Agent 根据此判断何时调用）',
    readOnly: true,  // true = 只读不修改系统，false = 有副作用

    inputSchema: {
      type: 'object',
      properties: {
        param1: {
          type: 'string',
          description: '参数描述',
          enum: ['option1', 'option2']  // 可选枚举
        },
        param2: {
          type: 'number',
          description: '数值参数'
        }
      },
      required: ['param1']
    },

    // run 是异步函数，接收解析后的参数对象
    run: async (args) => {
      // 实现逻辑...
      // 返回值会被 Agent 读取
      return JSON.stringify({ result: '...' });
    }
  }
];
```

### 6.4 Slash 命令格式

每个 `.md` 文件即为一个命令，文件名即命令名：

```markdown
<!-- commands/inspect-report.md -->
---
description: 巡检报告生成
---

你是一个巡检报告专家。按照以下步骤工作：

1. 确认巡检类型、区域、数据来源
2. 读取并分析巡检数据
3. 使用 recall_memory 搜索历史记录
4. 生成结构化巡检报告
...
```

### 6.5 系统提示词

`prompt.md` 内容会被追加到 Direct 引擎的基础系统提示词之后。用于：
- 注入领域知识（法规、分类、术语）
- 定义工具使用指引（何时调用哪个工具）
- 设定安全原则和语言风格

### 6.6 开发调试流程

```
1. 复制 plugins/examples/echo/ → plugins/my-plugin/
2. 修改 plugin.json 和 index.js
3. 启动应用: npm run dev
4. 在设置 → 插件页面点击「重载」
5. 查看是否加载成功（卡片正常显示 + 无错误标记）
6. 在对话中测试工具调用
7. 点击插件卡片查看详情确认信息正确
```

> 📌 **安全提醒**: 插件工具仅在 Direct 引擎下可用。实际飞行前须以当地空管部门和民航局最新规定为准，本插件信息仅供参考。
