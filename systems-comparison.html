Agent Systems · 2026

三套 Agent 系统的优缺点与 Token 消耗

Claude Code(CC-rev 重构)· Direct(KinetAios)· Codex —— 同一 ReAct 骨架,差异在执行安全、上下文管理与成本三条副线。

优缺点对比

Claude Code

最完备的 agent 抽象

优点

  • 五级上下文压缩:autocompact + microcompact + snip + collapse + reactive,长任务信息不丢
  • prompt cache 最强:静态/动态分段 + DYNAMIC_BOUNDARY + cache_control
  • subagent 全家桶:递归 query + run_in_background + team / swarm
  • 工具最全(数十个:Bash / Edit / Agent / Task…),流式边执行

缺点

  • 抽象最重:泛型 Tool<I,O,P> + Zod + 巨型 ToolUseContext,上手/维护成本高
  • Claude 模型单价高($3 / $15+ per 1M),$ 成本最贵
  • 代码量大,定制改造门槛高

Direct · KinetAios

最精简的同构骨架

优点

  • 与 CC 概念同构但去尽复杂度,~500 行核心,易读易改
  • GLM 直连 + OpenAI↔Anthropic 双协议,模型选择最灵活
  • 启动扫描用户环境(skills / MCP / CLI 预检),首次请求即准 → 减少试错重试,省 token
  • 工具精简(9 个)+ system 紧凑,每轮 schema 小 —— 实测 token 比 CC/Codex 少
  • 补齐后压缩 / 并行+subagent / 重试 / prompt cache 全有

缺点

  • 无流式边执行(先收完 tool_calls 再跑);压缩层级少(两级 vs CC 五级)
  • 无 fallback model;token 估算粗糙(length×0.6)
  • 无 OS 级沙盒(靠 shell 确认桥拦截)

Codex

最硬核的安全执行工程

优点

  • 唯一 OS 级沙盒:mac seatbelt / linux landlock+bubblewrap / win 受限令牌
  • Guardian 自动审查 + 批准→沙盒→拒绝则升级重试
  • Responses API + WebSocket + 粘性路由;远程 /responses/compact + WorldState 增量
  • prompt caching(comp_hash)+ 多 agent(继承式)

缺点

  • Rust 多 crate 最重(~90 crate),构建/贡献门槛最高
  • 无通用 model fallback
  • 行为在工具层非 prompt 层,调试靠工具日志而非提示词
  • OpenAI 单价中高
Token 消耗

实测视角(用户测:Excel 分析数据 → HTML)

同样任务下,Direct 的 token 消耗实测比 CC、Codex 都少 —— 推翻了纯机制"压缩强就省"的推断。

Direct实测最省 · 工具少 + system 紧凑 + 扫描减重试
Claude Code较多 · 数十工具 schema + 多层 system + thinking/frc 等额外 block
Codex较多 · 模型专属 prompt + 多 agent 开销

但任务足够长时,机制会反超

轮次极多时,CC 的五级压缩 + 强 cache 把旧消息摘要掉、每轮更轻;Direct 两级压缩会先丢细节 —— 长任务 CC 可能追上反超。中短任务里 Direct 的精简优势主导。

🔎

扫描减重试(Direct 独有)

启动扫描 skills / MCP / resolveBin 预检 CLI,首次请求就知道有哪些工具可用 —— 不会因"工具缺失 / 配置错"失败重试(失败重试 = 双倍 token)。CC/Codex 是被 spawn 的 CLI,没这层聚合预扫描。

📦

精简省每轮

Direct 9 个工具 vs CC 数十个;system 一段紧凑中文 vs CC 多层 dynamic sections(memory/scratchpad/frc…)。每轮的 tools schema + system 都更小,轮次越多累积越省。

⚖️

同模型下看效率

怎么选

三者 ReAct 骨架完全一致,差异全在执行安全、上下文管理、成本三条副线。没有"最好",只有"最合适你的约束"。

要完备 + 预算足 → Claude Code 要省钱 + 易改造 → Direct 要安全执行 → Codex
