三引擎架构

每个会话可独立切换引擎，切换时清除跨引擎上下文。三种引擎共享统一的 AgentEvent 事件流。

自研引擎

Kaios (Direct)

内置 ReAct 循环，直接调用 LLM API。唯一使用全部内置工具的引擎，支持子 Agent 派发、沙箱模式、上下文压缩。通过 OpenAI 兼容协议接入智谱 GLM、DeepSeek、通义千问、Ollama 等。

CLI 集成

Claude Code

spawn claude -p --output-format stream-json，解析 NDJSON 流，通过 --resume 恢复会话，注入记忆和项目规则到 system prompt。

CLI 集成

Codex

spawn codex exec --json，解析 JSONL 流，支持 resume 和 sandbox 模式。记忆和规则注入到 prompt。

跨平台 CLI spawn：npm-global CLI 在 Windows 上是 .cmd shim，Node 拒绝直接 spawn（CVE-2024-27980）。引擎自动将 .cmd/.bat 走 shell:true，直接 .exe 走 clean argv。还会增强 PATH，让 GUI 启动的 Electron 找到全局安装的 CLI。

🔧

内置工具（Direct 引擎）

9 个工具，只读工具可同轮并发执行，写工具串行。沙箱模式可限制写操作范围。

工具类别功能
shell写执行 shell 命令（Windows 走 cmd.exe），执行前请求用户确认，支持 AbortSignal 中断
read_file只读读取 UTF-8 文本文件，自动检测编码
write_file写写入文件（覆盖），无长度限制，写前自动快照
edit_file写精确替换文件中的片段，支持 replace_all，写前自动快照
grep只读正则搜索工作目录，返回文件:行号:内容，自动排除 node_modules/.git
glob只读按 glob 模式列出文件，返回相对路径
git_diff只读查看 git diff（工作区/staged/指定 ref），比 shell git diff 更干净
web_fetch只读抓取 HTTP(S) URL 的文本内容（GET）
recall_memory只读语义搜索历史记忆（FTS5 全文检索）
dispatch_agent特殊派发子 Agent 执行只读子任务，支持指定引擎（direct/claudeCode/codex）

沙箱模式：readOnly 拦截所有写工具 → workspaceWrite 限制写入在 cwd 内 → fullAccess 完全开放。Claude Code 和 Codex 引擎通过各自的 flag（--permission-mode / -s）传递沙箱级别。

跨引擎流水线

将多个 AI 引擎串联成 pipeline，上一步输出自动传入下一步。

示例： Claude 分析架构 → Codex 实现代码 → GLM 审查质量 步骤配置： ├─ 步骤 1: [Claude Code] "分析项目架构,给出重构方案" ├─ 步骤 2: [Codex] "根据以下方案实现代码: {step1_output}" └─ 步骤 3: [Kaios] "审查以下代码改动,指出问题: {step2_output}"

多机协作（MCP）

MCP 协议（Model Context Protocol）— 把多台机器组成 Agent 协作网络。

本机 Server

📡 Local MCP Server

把自己暴露为远程工具节点。开启后局域网内其他机器可通过 SSE transport 连接，调用本机工具（shell / read_file / web_fetch 等）。支持端口和 Token 认证。

远程 Client

🔗 Remote MCP Client

在设置中添加远程 MCP Server 列表。SSE transport（POST JSON-RPC），支持 Token 认证。连接成功后工具自动暴露给 Direct 引擎。

渐进连接：后台异步连接所有 server，每连好一个就加入工具池（失败静默跳过）。远程节点支持后台重连（SSE 最多重试 5 次，超时 10 分钟）。

🔒 安全边界

contextIsolation: true + nodeIntegration: false。Renderer 零 Node 访问，只能调 Preload 暴露的 KinetAPI。

📦 SQLite + FTS5

better-sqlite3 同步引擎。FTS5 虚拟表驱动全文检索；conversations + turns 表存会话状态；memories 表存长期记忆。WAL 模式。

🔄 统一事件流

所有引擎发同样的 AgentEvent（token / tool / cost / status / done / error）。applyEvent() 是纯函数，Main 和 Renderer 各调一次，改一处生效。

🧩 Shell 确认桥

Main 不能弹 UI → 发 confirm-request 到 Dashboard → Renderer 显示模态 → 回传 confirm-response。approval: 'never' 可跳过。
