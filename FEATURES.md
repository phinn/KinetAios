## 三、可加功能清单(按价值 × 难度)

> 价值 ★(越高越值得)/ 难度 ◆(越高越费工)。建议优先「价值高、难度低」的。

### A. 代码上下文 —— 当前最大短板

| # | 功能 | 对标 | 价值 | 难度 | 说明 |
|---|------|------|:---:|:---:|------|
| A1 | **@文件/@符号 引用** | 全员 | ★★★★★ | ◆◆ | composer 里 `@path` 把文件/符号内容注入上下文。最常用的上下文方式,比附件更精准 |

| A3 | **代码库索引(语义检索)** | Cursor/Claude(CodeGraph) | ★★★★ | ◆◆◆◆ | Embedding + 向量库,「这个功能在哪实现」类问题。重;可接 MCP(codegraph 已有路径) |

## 四、优先级建议

### 短期(高性价比,几天内可做)
- **D5 托盘图标 + 真全局热键** —— 体验第一缺口,纯 Electron,半天
- **G1 项目规则文件(AGENTS.md)** —— 读个文件注入,极简但价值高
- **A2 grep/glob 内置工具** —— 几十行,Direct 引擎立刻好用
- **B1 edit_file diff 工具** —— write_file 的安全升级,带确认
- **A1 @文件引用** —— composer 解析 `@path` 注入,中等工作量但最高频

### 中期(差异化,1-2 周)
- **E1 图片输入** —— 多模态消息改造
- **C1 TODO 清单** —— 复杂任务可视化
- **F1 Git 工具** —— 原生版本控制
- **D1 命令面板** —— 键盘流体验
- **G6 Prompt 缓存** —— 成本/速度
- **G3 细粒度权限** —— 安全上台阶

### 长期(重投入,月级)
- **A3 代码库索引/语义检索** —— 接 codegraph MCP 是捷径
- **B2 编辑检查点/回滚** —— 文件变更追踪
- **C2 子 agent** —— 架构级
- **F4 VSCode 插件** —— 独立工程
- **D4 内联补全** —— 要嵌编辑器

## 五、差异化定位建议

KinetAios 现在的差异化点:**多引擎统一面板(Direct + Claude Code + Codex 切换)+ 每会话独立模型 + 本地优先(SQLite/MCP)**。

建议强化方向(别和 Cursor 拼编辑器,那打不过):
1. **「编排层」定位**:一个面板调度多家引擎 + MCP + skills,做「跨引擎对比/接力」(同一问题问三个引擎,或 Direct 规划、Claude 执行)。
2. **本地/隐私**:所有数据本地,不强制账号。强化离线模型(Ollama 接入)。
3. **工作流自动化**:hooks + 自定义命令 + 计划模式,做「可重复的任务流水」。
