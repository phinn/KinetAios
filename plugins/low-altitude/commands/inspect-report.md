你现在要执行巡检报告生成流程。请按以下步骤操作:

## 1. 确认巡检信息

向用户确认:
- 巡检类型(电力线路 / 管网 / 光伏板 / 风电机组 / 桥梁 / 铁路等)
- 巡检区域和日期
- 数据来源(本地照片/视频文件夹路径, 或口述发现的缺陷)
- 无人机/传感器型号(如有)

## 2. 数据处理

根据用户提供的数据来源:

### 如果有本地图片/视频文件夹:
- 使用 `read_file` 读取巡检日志或记录文件
- 如果图片需要批量分析,可以用 `shell` 调用本地推理脚本(如果有的话)
- 用你的视觉能力(如果模型支持)分析用户贴入的巡检照片

### 如果是口述/文字描述:
- 直接整理用户提供的缺陷信息

## 3. 历史对比

- 使用 `recall_memory` 搜索该区域/线路的历史巡检记录
- 对比上次巡检结果, 标注新增缺陷和已修复项

## 4. 生成报告

输出结构化巡检报告:

```markdown
# 巡检报告

## 基本信息
- 巡检类型:
- 巡检区域:
- 巡检日期:
- 操作员:
- 设备:

## 巡检结果摘要
- 发现缺陷总数: X 处
- 严重等级分布: 紧急 X / 重要 X / 一般 X
- 与上次巡检对比: 新增 X / 已修复 X / 恶化 X

## 缺陷明细
(按严重等级排列, 每条含: 位置、描述、照片编号、建议措施)

## 历史趋势
(该区域缺陷数量变化趋势)

## 建议与行动计划
- 紧急处理项: ...
- 计划性维护: ...
- 下次巡检建议: ...
```

## 5. 可选: 导出文件

询问用户是否需要将报告保存为文件(`write_file` 生成 Markdown 或 CSV)。
