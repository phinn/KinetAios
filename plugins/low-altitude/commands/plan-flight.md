你现在要执行一个完整的飞行任务规划流程。请按以下步骤操作:

## 1. 收集任务信息

向用户确认以下信息(用户消息中已有的不用再问):
- 起飞点坐标(纬度/经度)
- 降落点坐标(纬度/经度)
- 途经点(如有)
- 计划飞行高度(米)
- 计划飞行时间(大致时段)
- 无人机型号/类别
- 飞行场景(巡检/物流/航拍/植保/测绘等)

## 2. 并行查询

使用以下工具**并行**执行(这些是只读工具,可以同时调用):
- `weather_brief` — 查起飞点和航线中点的气象
- `airspace_check` — 查起飞点和降落点的空域状态
- `compliance_check` — 做法规合规检查

## 3. 航线规划

使用 `flight_plan` 工具计算航程、飞行时间、电量需求。

## 4. 综合评估

根据查询结果给出:
- **气象评估**: 是否适合飞行?有哪些风险时段?
- **空域评估**: 是否在限飞区?需要什么许可?
- **合规评估**: 需要办理哪些手续?
- **航线评估**: 电量是否充足?有无优化建议?

## 5. 生成任务书

将所有结果整理成结构化的飞行任务书(Markdown 格式),包含:
- 任务概述
- 气象简报
- 空域状态
- 合规检查结果
- 航线详情(含航点 CSV)
- 风险提示与建议
- 最终结论(建议飞行 / 有条件飞行 / 不建议飞行)

如果任何环节存在阻断性问题(如限飞区、恶劣天气),明确给出 **不建议飞行** 的结论并说明原因。
