CC-rev 的差异化

工程完备度最高 + prompt-cache 友好的分段 system prompt + subagent 全家桶。抽象最重(Tool<I,O,P> 泛型 + Zod + 巨型 ToolUseContext)。

Direct v2 的差异化

概念同构的最小实现,骨架与 CC 一致但去尽复杂度;独有的 GLM 直连 + 协议双向。v2 补齐三块后核心能力已追平,只是深度浅。

Codex 的差异化

唯一带 OS 级沙盒执行 + Guardian 自动审查 + WebSocket 传输 + 远程 compact。Rust 多 crate 架构最重,把"安全执行"做成一等公民。

一句话定位:CC-rev = 最完备的 agent 抽象;Direct v2 = 最精简的同构骨架(够用、可读、易改);Codex = 最硬核的安全执行工程。三者 ReAct 骨架完全一致,差异全在执行安全、上下文管理、子任务三条副线。
